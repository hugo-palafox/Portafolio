# Codebase Hygiene

## Use this skill when
- Adding new components, models, services, or pages.
- Refactoring existing code.

## Rules
- Use consistent naming and folder conventions.
- Prefer small reusable components over large files.
- Remove duplication when adding new features.
- Avoid premature abstractions.
- Prefer simple maintainable implementations.

## Validation checklist
- Is new complexity justified?
- Can another engineer find things quickly?
- Did this change reduce or increase duplication?
