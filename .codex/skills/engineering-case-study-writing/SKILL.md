# Engineering Case Study Writing

## Use this skill when
- Writing project summaries, detail sections, and technical storytelling content.

## Preferred structure
- Problem
- Architecture
- My Role
- Technical Challenges
- Solution
- Outcome
- Lessons Learned

## Rules
- Use technical language and realistic scope.
- Explain tradeoffs, integration points, and deployment constraints.
- Avoid buzzwords and marketing-style claims.
- Keep summaries concise and specific.
- Emphasize system ownership and measurable impact.
