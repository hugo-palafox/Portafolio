# Accessibility and UX Safety

## Use this skill when
- Creating or updating any UI component or page.

## Rules
- Use semantic HTML structure and heading hierarchy.
- Ensure visible keyboard focus states.
- Keep contrast strong in dark theme.
- Do not rely on color alone to communicate meaning.
- Respect reduced-motion preferences.

## Validation checklist
- Can all primary actions be reached by keyboard?
- Are headings ordered and meaningful?
- Are focus states and contrast clearly visible?
