# Tailwind Architecture

## Use this skill when
- Building or refactoring Tailwind-styled components.
- Repeated class sets appear across pages/components.

## Rules
- Avoid long repeated class strings in multiple files.
- Extract repeated patterns into reusable component styles.
- Keep responsive breakpoints and spacing usage consistent.
- Prefer semantic component classes for complex elements.

## Validation checklist
- Is markup readable without scanning long utility class strings?
- Are repeated patterns extracted instead of duplicated?
- Do responsive rules follow one consistent pattern?
